Objetivo: analizar volcados de la RAM para obtener clave para poder desbloquear el BitLocker

En primer lugar, se analiza el archivo .vbox para detectar vulnerabilidades (como tenían BitLocker sin pin era atacable, y al no tener PAE/NX (y mas cosas) era "sencillo"). Luego se utilizaron herramientas para intentar extraer la clave y montar con ella el disco en Kali Linux (o en Windows como un disco externo usando una herramienta), pero no se consiguió. La más destacable es Volatily3, el prbolema fue que no hay plugin específico para estar versión de Windows, por lo que no funcionaba.

Luego ya se optó por la forma "manual" mediante scripts de python y apoyandose (para tener referencias visuales) en la app HxD. Se comenzó buscando tags (como FVEK), cogiendo los 64 bytes posteriores (creo que eran 64, sino lo pondrá en el informe) y probando a montar el disco con todas las obtenidas. Como no funcionaba, se optó por coger ventanas de bytes posteriores a los tags (ej. 1024 bytes) y extraer todas las cadenas de 64 bytes (o del tamaño que fuese). El problema es que eran muchas para probar por fuerza bruta a montar el disco (muchos falsos positivos, para no perder los verdaderos positivos), por lo que se siguieron heurísticas de filtrado. Principalmente estaban basadas en entropía (la cadena objetivo tenía que tener cierta aleatoriedad), realizando análisis estadísticos (por ejemplo, chi cuadrado). Aún así eran demasiadas por lo que se utilizo doble volcado (y creo que llegué a usar triple volcado). Se hacía los métodos anteriores (examinar ventanas posteriores a los tags y filtrar con métodos estadísticos), y como la cadena tendría que estar en los tres volcados (porque la necesita el vTPM) quedarse solo con las cadenas que estuviesen en los tres volcados. Creo recordar que en este punto el número de candidatos fue manejable y por fuerza bruta probé a montar el disco, pero ninguna funcionó.
